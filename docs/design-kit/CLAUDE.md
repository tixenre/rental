## Stack

- **Framework**: Next.js (App Router) + React 19
- **Styling**: Tailwind v4 вЂ” tokens live in `src/kit/tokens.css`
- **Fonts**: TT Commons (primary, all weights) + Champ Black (display/hero only)
- **Icons**: Lucide React вЂ” import individually, never the barrel. `strokeWidth={1.5}` always.
- **Component lib**: `src/kit/components/` вЂ” 10 presentational components, no store

## Mobile вЂ” critical rules

```css
/* 1. viewport-fit=cover in <head> вЂ” always */
/* <meta name="viewport" content="width=device-width, initial-scale=1, viewport-fit=cover"> */

.bottom-bar    { padding-bottom: calc(var(--space-4) + env(safe-area-inset-bottom)); }

/* 4. Input zoom prevention вЂ” always */

/* 5. Drawer overscroll */

/* 6. Use dvh not vh */
```

**Touch targets:** minimum 44Г—44px on all interactive elements. Use padding or `::before` pseudo-element to extend hit area on small chips.

**Thumb zone:** Primary CTAs (Reservar, Confirmar) в†’ bottom 50% of screen. Never put critical actions in top 22%.

## Patterns to never break

1. **Never use a Google Font substitute for TT Commons or Champ Black** вЂ” fonts ship locally in `kit/fonts/`
2. **Champ Black is for display text only** вЂ” not headings, not UI labels
3. **Never hardcode hex colors** вЂ” always use CSS tokens (`var(--amber)`, `var(--ink)`, etc.)
4. **Always `strokeWidth={1.5}` on Lucide icons**
5. **Import Lucide icons individually** вЂ” never `import * from "lucide-react"`
6. **All prices through `formatARS()`** вЂ” never format manually
7. **`dvh` not `vh`** for full-height elements on mobile
8. **`viewport-fit=cover` + safe area insets** on every screen that has a sticky topbar or bottom bar
