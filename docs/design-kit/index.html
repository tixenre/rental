01 · Quick start

Instalación en 3 pasos

Pegale esto a Claude Code en cualquier proyecto Tailwind v4 + React, o seguilos a mano.

kit/INSTALL.md
1

Copiá el kit

Mové la carpeta kit/ a src/kit/ de tu proyecto. Las fuentes vienen adentro.

# desde la raíz de tu proyecto
cp -r path/to/kit src/kit
2

Wireá los imports CSS

En src/styles.css, justo después de Tailwind:

@import "tailwindcss";

@import "./kit/fonts.css";
@import "./kit/tokens.css";
@import "./kit/recipes.css";
3

Instalá las deps

Lo que los componentes necesitan más allá de Tailwind:

npm install clsx tailwind-merge \
  class-variance-authority \
  @radix-ui/react-slot lucide-react
02 · Qué hay en la caja

Estructura del kit

Tres archivos CSS de base + componentes React tipados. Cada pieza es independiente, no hay barril único.

kit/ ├── fonts/ tt commons + champ black ├── fonts.css @font-face ├── tokens.css @theme + :root ├── recipes.css .t-* .wordmark .grain ├── index.css atajo de los 3 ├── INSTALL.md ├── lib/ │ └── format.ts formatARS, ranges ├── types/ │ └── equipment.ts Equipment, EstadoPedido └── components/ ├── lib/ │ └── cn.ts ├── button.tsx ├── input.tsx + Search + Label ├── addon-pills.tsx ├── estado-badge.tsx ├── equipment-card.tsx ├── price-block.tsx ├── view-toggle.tsx ├── empty-state.tsx ├── stat-card.tsx └── topbar.tsx
tokens.css El corazón. Bloque @theme inline con colores, radii, type stacks. Tailwind v4 lo lee y genera utilities: bg-amber, text-ink, border-hairline.
fonts.css TT Commons + Champ Black vendoreadas localmente. Las del manual oficial de marca, no las free de Google.
recipes.css Utilities compuestas que no son tokens: type recipes (.t-display-1, .t-h1, .t-eyebrow), .wordmark, .grain, .bg-amber-tape, safe-area helpers.
components/ Diez piezas presentacionales, una por archivo. Tipadas, con JSDoc + ejemplo. No traen store; recibís estado por props.
lib/ + types/ Capa chica de utilidades. formatARS, formatRentalRange, jornadaLabel + tipos compartidos Equipment, AddonItem, EstadoPedido.
