/* El Aplicar se habilita solo si el rango es 100% válido */
const blocked = range.some(d => isClosed(d) || hasNoStock(d, cartItems));
const extraJornada = endTime > startTime;   // devolver más tarde que retiro

const canApply = start && !blocked;          // jornada extra NO bloquea: avisás, no impedís

/* Tres niveles de feedback, nunca dos a la vez */
{blocked        && <Bar variant="error">…</>}   // destructive · bloquea Aplicar
{extraJornada   && <Bar variant="warn">…</>}    // amber · informa, deja seguir
{!start         && <Bar variant="muted">…</>}   // hint inicial
Jerarquía de feedback (una sola por vez): error --destructive > warning --amber > hint muted. El error bloquea Aplicar; la advertencia de jornada extra no — avisás y dejás decidir. · Vacío: retiro con borde dashed + placeholder, devolución oculta (no existe sin retiro), Aplicar disabled, hint muted. · Jornada extra: la card de devolución vira a --naranja y muestra +1 J; el footer cuenta 4 J. La copy ofrece la salida concreta (≤ 18:00). · Día cerrado: día tachado + marcado en destructive; viene de useHorarios. · Sin stock: el día dentro del rango se marca con punto destructive; el error nombra qué equipo y qué día, y da dos salidas. · Por qué validar en cliente si el backend igual rechaza: feedback inmediato sin round-trip — el usuario corrige antes de mandar.
