/* App.tsx — un solo Toaster, abajo a la derecha */
<Toaster
  position="bottom-right"
  offset={16}
  toastOptions={{
    classNames: {
      toast: "rounded-[10px] border-hairline shadow-lg bg-surface-elevated",
      title: "font-sans text-[13px] font-semibold text-ink",
      description: "text-[12px] text-muted-foreground",
    },
  }}
/>

/* Llamadas — voz de marca: vos, lowercase de sistema, punto final */
toast.success("Pedido #1247 enviado", {
  description: "Te llevamos a tu portal para seguir el estado.",
  duration: 4000,
});

/* Undo — el patrón canónico para acciones destructivas del carrito */
toast("Sony a7s III quitada del carrito", {
  action: { label: "Deshacer", onClick: () => restoreItem(id) },
  duration: 6000, // más tiempo: la acción es reversible
});
4 variantes + neutral: success --verde · info --azul · warning --naranja · error --destructive · neutral (undo) sin color. La barra de acento de 3px a la izquierda es la única señal cromática — el resto queda en bone+ink. · Duración por gravedad: success 4s · info/warning 5s · undo 6s (reversible, dale tiempo) · error es Infinity hasta que el usuario lo cierre o reintente. · Acción, no link: "Reintentar" / "Deshacer" como botón a la derecha. Una sola acción por toast. · Copy: primera persona, específico — "No pudimos guardar el pedido", nunca "Error 500". · Nunca para confirmaciones bloqueantes — eso es un modal. El toast es para lo que ya pasó.
