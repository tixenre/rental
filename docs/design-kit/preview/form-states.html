Implementación
/* Focus ring — siempre amber, 3px, amber-soft fill */
.inp:focus {
  border-color: var(--amber);
  box-shadow: 0 0 0 3px var(--amber-soft);
}
/* Error — mismo grosor, tinte destructive */
.inp.err:focus {
  box-shadow: 0 0 0 3px color-mix(in oklch, var(--destructive) 13%, transparent);
}

/* React Hook Form — validar en onBlur, no onChange */
useForm({ mode: "onBlur" })

/* Error message bajo el campo, nunca tooltip */
{errors.email && (
  <FieldError>{errors.email.message}</>
)}
Validar en onBlur — no en onChange (reduce ruido mientras el usuario escribe) ni solo en submit (demasiado tarde). Excepción: búsqueda libre siempre en onChange. · Copy en primera persona, específico: "Ingresá un correo válido." — no "Error", no código HTTP. Terminá con punto. · Ícono en el helper de error (circle-warning) para que el mensaje sea accesible más allá del color. · Campo requerido: marca amber * junto al label, nunca al final del formulario. Opcional (se aclara la excepción, no la regla). · Form-level error arriba del formulario, antes del primer campo — nombra específicamente qué salió mal y ofrece salida. · Loading remoto (ej. validar código de referido): skeleton sobre el input, no spinner dentro. · Disabled ≠ no existe — siempre se ve el campo, aunque no sea editable. Nunca lo ocultes.
