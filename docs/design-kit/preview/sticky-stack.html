Implementación
/* Cada barra "se pega" justo debajo de la anterior.
   El top acumula las alturas de las barras de arriba. */
.topbar      { position: sticky; top: 0;    z-index: var(--z-topbar); }      /* 56px */
.cart-strip  { position: sticky; top: 56px; z-index: var(--z-cart-strip); }  /* 36px */
.category-bar{ position: sticky; top: 92px; z-index: var(--z-cat-bar); }      /* 48px */
.sub-toolbar { position: sticky; top: 140px;z-index: var(--z-sub-toolbar); }

/* CartStrip solo se monta si hay ítems. Su altura se expone
   como var para que las barras de abajo ajusten su top. */
:root { --cart-strip-h: 0px; }           /* JS la setea a 36px al montar */
.category-bar { top: calc(56px + var(--cart-strip-h)); }
CartStrip (z 45) aparece solo con ítems en el carrito; va en --ink con dot amber, muestra cantidad + ventana de fechas y CTA "Ver pedido". Es el puente entre el topbar y el contenido — no se confunde con el CartMiniBar (ese es bottom-fixed en mobile). · CategoryBar (z 40) tabs con scroll horizontal (mismo overflow que el chip rail); activo en --surface + w600. El count en mono. · El top se acumula: cada barra se pega bajo la anterior sumando alturas. Cuando la CartStrip se monta, --cart-strip-h empuja a la CategoryBar hacia abajo. · sub-toolbar (z 30) solo en modo lista. · Nunca inventes z-index — usá los nombres de la escala. El drawer (61) siempre tapa todo esto.
